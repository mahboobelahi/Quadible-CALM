On FASTory Line following actions are perfored in order to finish drawing of a cell phone model:

  1. 1. The paper loading operation is done by robot which takes certain time which cannot be reduce.
  2. 2. Pallet movement on bypass and production conveyors is fixed and cannot be minimized.
  3. 3. Loading and unloading of pallet is manual and depend on the efficiency of pallet loading operator which is not reducible.
  4. 4. The paper loading operation is done by robot which takes certain time which cannot be reduce.
  5. 5. There are two tasks performed by production robots, changing of pen cliours and drawing of cell phone recipes. There is a possibility to reduce/minimize or optimize the job time by assignment of tasks though these operations need certain time which is not reducible, but the task and cycle time optimization is achieved by assigning the combination of different task to these work cells.

Author developed three different production policies to optimize the average cycle time of working cell by assignment of different combination of jobs to production cells. The name of these production policies is listed below:

  1. 1. Available Free workstation (Reference Policy)
  2. 2. Fixed Colour Policy
  3. 3. Fixed Drawing Recipe and Colour Policy

Available Free workstation (Reference Policy)

This is the novice production policy which does not balance assembly line. Orchestrator takes this policy and start production without any balancing task assignment. This reference policy is good to measure the time taken by work cells to draw test case cell phone production model. In reference production policy each free WC is assigned a whole unit of cell phone model is assigned to draw. Here pallet left the production zone when all the components are drawn on the same work cell. Figure shows the precedence diagram of test model for the reference production policy.

Fixed Color Policy

Before discussing second policy, please note that in this process there is a repetative task which is pen change. The pen change job involves pick and place type operations. These operations are time consuming and slow down the drawing process furthermore, pen change task unnecessarily increase the work element (N) hence the task time also increases so it is unnecessary to include them in work element. In policy 2 each workstation is assigned a specific colour out of available drawing colours (Red, Green and Blue) as FASTory line is capable to draw 729 different cell phone models based on the different combinations of RGB colours and 9 distinct components so fixing the colours do not harm the production of any product mix combination. In fixed colour policy orchestrator assigned a pen change task to the workstations at start- up phase hence robot of every workstation has assigned colour in its gripper before the start of production. Hence this task assignment does not include in calculating the ALB parameters for this policy. Figure 44 shows the sequence diagram of policy 2. When production starts, order is placed on the pallet and pallet goes to the zone 1 of the WC and orchestrator check whether the workstation is capable to draw the component in specific colour or not. If it can than production starts otherwise it bypasses that station. Colour fixation technique significantly reduces total task time as well as work elements from 7 to 4. Figure shows the precedence diagram of policy 2 work elements.

In policy 2, orchestrator allocate jobs to nearest free work cell which is capable to perform drawing task i.e. colour of order component and workstation are same. In this policy each WC will perform only drawing task. During production most of the time each free workstation draws only one component hence the workstation operative time does not exceed from cycle time

Fixed Colour and Fixed Recipe Policy

This policy is slight modification of policy 2. As name of the policy indicates that beside fixing the colours of workstations it also fixes the drawing recipes.

The assignment of task done by orchestrator at run time during instantiation of workstation server class objects. Task assignment is such that:

  1. All types of frames (Draw1, Draw2, Draw3) of Red colour are drawn by cell 2.
  2. All types of screens (Draw4, Draw5, Draw6) of Green colour are drawn by cell 3.
  3. All types of keypads (Draw7, Draw8, Draw9) of Blue colour are drawn by cell 4.
  4. This pattern continues up till work cell 11 as listed in table 18.
  5. Work cell 12 to “ERROR” so orchestrator knew about the current state of work cell and bypass that work cell during production. This is done purposely to show that workstations under maintenance or error state can be handled by orchestrator.
